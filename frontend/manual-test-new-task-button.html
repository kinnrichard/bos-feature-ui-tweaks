New Task Button Refinement - Manual Test Checklist

Test URL: http://localhost:5173/jobs/test
Implementation Date: 2025-07-23
Features: Conditional positioning, hover states, mobile persistence

1. Empty Task List Tests

  • ☐ Navigate to a job with no tasks
  • ☐ Verify "New Task" button appears at the TOP of the list (not bottom)
  • ☐ Verify text shows in default color (gray/secondary)
  • ☐ Hover over the button - text should turn blue
  • ☐ Hover over the button - icon should change to blue variant
  • ☐ Hover over the button - label should remain visible (not hide)

2. Populated Task List Tests

  • ☐ Create at least one task
  • ☐ Verify "New Task" button moves to BOTTOM of the list
  • ☐ Hover anywhere in the New Task row
  • ☐ Verify text turns blue on hover
  • ☐ Verify icon switches to plus-circle-blue.svg
  • ☐ Verify label HIDES on hover (desktop only)
  • ☐ Move mouse away - label should reappear

3. Mobile/Tablet Tests

  • ☐ Open browser DevTools (F12)
  • ☐ Toggle device toolbar (Ctrl+Shift+M)
  • ☐ Select iPhone or iPad preset
  • ☐ Navigate to job with tasks
  • ☐ Verify "New Task" label is ALWAYS visible
  • ☐ Tap on the button - text should briefly turn blue
  • ☐ Verify label never hides on mobile

4. Icon Switching Tests

  • ☐ Inspect element to verify initial icon: /icons/plus-circle.svg
  • ☐ Hover over row
  • ☐ Verify icon changes to: /icons/plus-circle-blue.svg
  • ☐ Verify no animation/transition on icon switch (instant change)
  • ☐ Check Network tab - both icons should be preloaded

5. Responsive Behavior Tests

  • ☐ Start with desktop viewport (>768px)
  • ☐ Verify desktop hover behavior works
  • ☐ Slowly resize window to mobile width (<768px)
  • ☐ Verify label becomes persistent (no hiding)
  • ☐ Resize back to desktop - hover hiding should work again

6. Edge Cases

  • ☐ Create task, then delete all tasks - button should move back to top
  • ☐ Test with 50+ tasks - button should remain at bottom
  • ☐ Test rapid hover in/out - no flickering
  • ☐ Test keyboard navigation - Tab should focus the button
  • ☐ Test with touch + mouse device - appropriate behavior for each input

• Default text: #8E8E93 (--text-secondary)
• Hover/active text: #007AFF (--accent-blue)
